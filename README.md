# Sheets Simulator — обучающий тренажёр таблиц для 4–8 классов

Интерактивный prototype на **Next.js + TypeScript**: дружелюбный симулятор Excel/Google
Sheets, где школьник за 5 заданий учится реальным навыкам работы с таблицами и
основам анализа данных.

> Проект сделан как production-quality MVP для масштабной EdTech-платформы.
> Делается упор на сильное продуктовое мышление, понятный детский UX,
> реалистичный spreadsheet-experience и аккуратную архитектуру кода.

## Что это

Тренажёр, в котором ребёнок:

1. сначала проходит короткое введение (что такое таблица, строка, колонка, ячейка);
2. попадает в единый экран с настоящей spreadsheet-сеткой;
3. получает задания через task panel справа;
4. сначала **видит демонстрацию** действия, потом **повторяет сам**;
5. может нажать «Подсказка», «Проверить», «Сбросить», «Назад», «Дальше»;
6. видит progress bar по всему уроку, может перепрыгивать между заданиями;
7. учится на **реальных датасетах**: школьный буфет, кружки, библиотека.

Главный принцип: **«сначала покажи — потом дай попробовать самому»**.

---

## Кому это нужно

- школьники **4–8 классов** (в т.ч. с разным уровнем цифровой грамотности);
- учителя информатики и кружки;
- EdTech-платформы, которые хотят встроить модуль работы с данными.

UX рассчитан на ребёнка: дружелюбный, аккуратный, не «детсадовский»,
не сухой корпоративный, без перегрузки лишними кнопками.

## Архитектура

Решение чётко разделено на 4 слоя. Это позволяет добавлять новые задания
и виды проверок без правок UI и наоборот.

```
┌─────────────────────────────────────────┐
│  UI components                          │  src/components/
│  Spreadsheet, Toolbar, FormulaBar,      │
│  TaskPanel, ProgressBar, IntroScreen…   │
├─────────────────────────────────────────┤
│  Lesson orchestration                   │  src/components/lesson/LessonView.tsx
│  Координирует урок, шаги, демо,         │  src/lessons/checker.ts
│  кнопки, проверку, переходы.            │
├─────────────────────────────────────────┤
│  Spreadsheet engine (pure TS)           │  src/engine/
│  Типы, reducer, выделение, сортировка,  │
│  фильтр — без зависимости от React UI.  │
├─────────────────────────────────────────┤
│  Data + lessons (декларативно)          │  src/lessons/
│  Каждый урок = объект Lesson:           │
│  данные, шаги, демо, проверки.          │
└─────────────────────────────────────────┘
```

Ключевые принципы:

- **Pure engine.** `src/engine/` — это чистый TS без React. `spreadsheetReducer`,
  `runCheck`, `sortRows`, `filterRows` детерминированы и тестируются изолированно.
- **Декларативные уроки.** Каждый урок — это объект `Lesson` с массивом `LessonStep`.
  Шаг описывает: brief, hint, демо, способ проверки. Чтобы добавить урок,
  не нужно трогать UI.
- **Композиция проверок.** `LessonCheck` — discriminated union (`selectColumn`,
  `selectRow`, `selectCell`, `selectCells`, `highlightCells`, `sort`, `filter`,
  `choice`, `all`). Можно собирать сложные критерии из простых.
- **Demo + Highlight overlay.** Система туториала умеет одновременно (а) подсвечивать
  DOM-элементы (через data-tutorial-id) и (б) красить ячейки/строки/колонки
  прямо на таблице — на 2 секунды как пример.
- **Минимум зависимостей.** Только `next`, `react`, `framer-motion`, `lucide-react`,
  `clsx` и `tailwindcss`. Никакого тяжёлого state-менеджмента и UI-китов.

## 5 заданий

Каждое задание построено по сценарию **«объяснение → демо → действие → проверка»**.

### 1. Знакомство с таблицей
Датасет: продажи в школьном буфете.
- выделить колонку «Пирожки»;
- выделить строку «Среда»;
- выбрать ячейку «Пятница / Выручка».

### 2. Сортировка
- отсортировать «Выручка» по возрастанию;
- переключить ту же колонку на убывание (увидеть самый прибыльный день).

### 3. Поиск аномалии
В колонке «Пирожки» один день (Среда = 320) сильно выбивается из ряда 80–140.
- отсортировать по убыванию (аномалия выходит наверх);
- покрасить подозрительную ячейку красным;
- выбрать правильный вывод (multiple choice).

### 4. Поиск закономерности
Датасет: посещаемость кружков по дням недели.
- увидеть, что у шахмат стабильно высокая посещаемость во вторник и четверг;
- покрасить эти ячейки зелёным;
- выбрать правильный вывод.

### 5. Фильтр + вывод
Датасет: книги в школьной библиотеке.
- применить фильтр «Жанр = Фантастика»;
- по отфильтрованным данным определить самую популярную книгу.

Каждый шаг проверяется детерминированной функцией `runCheck(check, ctx)`,
которая принимает `LessonCheck` и состояние таблицы — и возвращает `{ ok, reason }`.
Reason пишем мягко: «Это другая ячейка. Сравни заголовок колонки и номер строки»,
а не «Неправильно».

## Как запустить

Требуется **Node.js 18+** и **npm**.

```bash
npm install
npm run dev
```

Открой [http://localhost:3000](http://localhost:3000).

Сборка production:

```bash
npm run build
npm run start
```

## UI/UX решения

- **Intro-экран** объясняет понятия колонка/строка/ячейка визуально, через мини-превью
  (а не словами). Это снимает страх «я не понимаю, что от меня хотят».
- **Реалистичная сетка**. Заголовки A/B/C сверху, номера строк слева, sticky headers,
  ресайз колонок drag'n'drop, hover/selected/active состояния, поддержка Shift-выделения
  диапазона, drag-selection мышью, навигация стрелками + Tab + Enter.
- **Inline-редактирование** (для незаблокированных колонок) запускается двойным кликом
  или по Enter — как в Sheets.
- **Toolbar** всегда виден, но содержит только то, что реально используется в уроках:
  сортировка, 4 цвета заливки, ластик, сброс. Без перегрузки.
- **FormulaBar** показывает адрес активной ячейки (`B3`) и её значение, как в Excel —
  даёт ребёнку якорь «что именно я выделил».
- **Task panel** справа содержит:
  - бейдж урока + шаг X из Y;
  - блок «Зачем это в жизни?» — связь с реальностью;
  - текущий шаг (brief);
  - подсказка / демо-баннер / feedback (мягкий, не наказательный);
  - кнопки: «Показать демо», «Подсказка», «Сбросить», «Проверить», «Дальше», «Назад».
- **Demo mode**. Нажав «Показать демо», ребёнок видит подсветку нужной кнопки
  (через highlight overlay) **и** короткую демонстрацию прямо на таблице
  (например, ячейка кратковременно подсвечивается синим). Через 2 секунды эффект
  откатывается, и ребёнок повторяет сам.
- **Прогресс**. Сегментированный progress bar сверху: каждый сегмент = урок,
  внутри — мини-прогресс по шагам. Можно перепрыгивать между уроками.
- **Celebration**. После урока/курса — мягкая модалка без «звёздочек» и пафоса:
  спокойное позитивное подтверждение и связь с реальностью.

Цветовая палитра — мягкая (бренд-голубой `#3b82f6`, янтарный для подсказок,
изумрудный для успеха, пастельные заливки для подсветок). Мультяшности нет.
Шрифт Inter, скругления 8–16px, тёплые тени.

## Расширение: как добавить новый урок

1. Создай датасет в `src/lessons/data/yourDataset.ts` (`ColumnDef[]`, `RowData[]`).
2. Создай `src/lessons/lessonN-yourtopic.ts`:

   ```ts
   import type { Lesson } from "./types";
   import { yourColumns, yourRows } from "./data/yourDataset";

   export const lessonN: Lesson = {
     id: "lesson-N-...",
     badge: "N",
     title: "...",
     subtitle: "...",
     goal: "...",
     reallifeNote: "...",
     columns: yourColumns,
     rows: yourRows,
     steps: [
       {
         id: "...",
         title: "...",
         brief: "...",
         hint: "...",
         demo: { callout: "...", highlightTargets: [...] },
         check: { kind: "selectColumn", colKey: "..." },
         successMessage: "...",
       },
     ],
   };
   ```

3. Добавь его в `src/lessons/index.ts` в массив `lessons`.

Никаких правок UI не требуется. `runCheck` уже умеет все основные проверки;
если нужна экзотическая — добавь новый вариант в `LessonCheck` и одну ветку в
`runCheck`.
